## 2. P0 — 真实 Bug（演示当场会被发现）

### Bug-1：顶栏用户头像取错字符
- **位置**：`src/layouts/MainLayout.vue:121`
- **现状**：`name.charAt(name.length - 1)` 取的是用户姓名 **最后一个字**
  - 登录用户 realName = "张管理" → 头像显示"理"
  - 顶栏文字 = "理 张管理"（视觉错位）
- **触发场景**：任何已登录页面，右上角用户区
- **建议改法**：
  ```js
  // MainLayout.vue:121
  const avatarText = computed(() => {
    const name = authStore.currentUser?.realName || ''
    return name.charAt(0) || '?'  // 改 length-1 → 0
  })
  ```
- **优先级**：高（任何用户登录都能看到）

### Bug-2：Settings 角色字段空白
- **位置**：`src/views/Settings.vue:25-33`
- **现状**：
  ```vue
  <el-input disabled>
    <template #default>  <!-- el-input 没有 #default slot -->
      <el-tag>{{ isAdmin ? '管理员' : '测试员' }}</el-tag>
    </template>
  </el-input>
  ```
  el-input 仅有 `#prefix` / `#suffix` / `#prepend` / `#append` 四个 slot，#default 内的内容被吞掉
- **触发场景**：登入后点击右上角头像 → 个人设置 → "角色"行空着
- **建议改法**：
  ```vue
  <el-form-item label="角色">
    <el-tag :type="isAdmin ? '' : 'info'" size="small" effect="plain">
      {{ isAdmin ? '管理员' : '测试员' }}
    </el-tag>
  </el-form-item>
  ```
- **优先级**：高（演示时演示者会想强调角色权限，结果是个空字段）

### Bug-3：BrokerHealthPanel 启动期 3 条 runtime compilation 警告
- **位置**：`src/components/execution/MqProbePanel.vue:120-130`
- **现状**：
  ```js
  const StatusBadge = {
    props: ['status', 'labels'],
    computed: { /* ... */ },
    template: '<el-tag :type="type" size="small" effect="plain">{{ text }}</el-tag>'  // 字符串模板
  }
  ```
  在 Vite + ESM build 下 runtime 不能编译 string template。打包后该组件会渲染失败。
- **触发场景**：打开 Connection 页 → 选中任意 Broker → 控制台 3 条 warning；dist 模式下三级检测卡 status badge 渲染为空白
- **建议改法**：改成 SFC 子组件（推荐）或 render function：
  ```js
  import { h } from 'vue'
  const StatusBadge = {
    props: ['status', 'labels'],
    setup(props) {
      return () => h('el-tag', { type: 'success', size: 'small', effect: 'plain' }, text)
    }
  }
  ```
- **优先级**：高（dev 下不显，但 dist 后必坏；现在 11 个页面是 mock 演示所以没人触发）

### Bug-4：el-radio `label` API 多处将弃用

## 3. P1 — 体验问题（影响操作效率）

### Issue-1：Connection 页面"被测系统"选择器与顶栏重复
- **位置**：`src/views/Connection.vue:22-29` + `src/layouts/MainLayout.vue:44-52`
- **现状**：顶栏已经有一个"被测系统"下拉，页面顶部"系统选择条"里又做了一个。同一份 store 双向同步，但 UI 上是 100% 重复，1366px 屏挤、1920px 屏浪费
- **建议改法**：
  - **方案 A（推荐）**：去掉页面内 select，仅在系统选择条里保留"负责人/模块数/Broker数/在线数"等只读 chip 信息
  - **方案 B**：把顶栏那个下拉删掉，因为页面内的更详细（含状态摘要）
- **优先级**：中（视觉清爽度）

### Issue-2：KnowledgeModel 文档行点击不展开
- **位置**：`src/components/report/KnowledgeBase.vue` 文档表格
- **现状**：点文档行无反应，必须点小箭头才能展开查看片段
- **触发场景**：用户期待"点哪行就展开哪行"，Element Plus table 默认行为正是这样
- **建议改法**：在 el-table 上加 `@row-click` 事件，调用 expand 切换
  ```js
  const expandedRows = ref([])
  function onRowClick(row) {
    const i = expandedRows.value.indexOf(row.id)
    if (i >= 0) expandedRows.value.splice(i, 1)
    else expandedRows.value.push(row.id)
  }
  ```
- **优先级**：中（违反用户预期）

### Issue-3：测试任务头行 5 个元素挤一行
- **位置**：`src/views/TestTask.vue:42-72`
- **现状**：任务名输入 + 状态 tag + 优先级 tag + 3 个操作按钮（提交执行/复制/删除）挤在一行
- **触发场景**：1366 屏按钮会换行，1920 屏虽然没换但任务名被压窄
- **建议改法**：把"状态/优先级/元信息（创建时间/更新于）"移到第二行（`task-head__meta`），第一行只保留"任务名输入 + 复制 + 删除 + 提交执行"
  ```vue
  <div class="task-head__row">
    <el-input v-model="editName" class="task-head__name" />
    <el-button @click="onDuplicate">复制</el-button>
    <el-button type="danger" plain @click="onDelete">删除</el-button>
    <el-button type="primary" @click="onSubmitExecution">提交执行</el-button>
  </div>
  <div class="task-head__meta">
    <el-tag>{{ statusLabel }}</el-tag>
    <el-tag>{{ priorityLabel }}</el-tag>
    <span>创建于 {{ createdAt }}</span>
  </div>
  ```
- **优先级**：低（视觉问题）

### Issue-4：协议页头部 6 元素横排
- **位置**：`src/components/protocol/ByteFieldTree.vue` 头部
- **现状**：协议名 input + 当前类型 select + 字节序 select + 导入/导出/保存/删除 4 个按钮 = 7 个
- **建议改法**：

  - 把"类型/字节序"做成 inline 可点击 tag（点一下弹下拉切）
  - "删除" 用 `<el-popconfirm>` 包起来，避免误删
- **优先级**：低

### Issue-5：执行编排底部"就绪检查"与"上一步/下一步"分离
- **位置**：`src/views/Execution.vue:88-129`
- **现状**：footer 与 PlanTable 内的就绪检查块视觉断开，footer 仅用 tooltip 提示"为什么不能继续"
- **建议改法**：把"就绪检查通过/未通过"作为 footer 顶部一个色条（绿/红），"未通过"时列出阻塞项文字
- **优先级**：低

## 4. P2 — 演示连贯性（演示最容易穿帮的环节）

### Issue-6：Dashboard 与执行页 / 异常页没有实时联动
- **位置**：Dashboard ↔ execution store / exception store
- **现状**：
  - 执行完成后，Dashboard 的"最近任务"列表**不会自动插入到顶部**，但 store 里其实有这条新 run
  - 异常处置后，Dashboard 异常 badge（待处理 8）**不会随状态变化实时更新**
  - 演示时演示者要刷新页面才能看到变化
- **建议改法**：写一个轻量 pub-sub（mitt 或 EventTarget）让 store mutation 触发跨组件刷新
  ```js
  // src/utils/bus.js
  import mitt from 'mitt'
  export const bus = mitt()
  bus.on('task:run-finished', () => { /* dashboard 重新拉数据 */ })
  ```
  在 `useExecutionStore` 的 `end()` 里 `bus.emit('task:run-finished', run)`
  在 `useDashboard` 的 `onMounted` 里 `bus.on('task:run-finished', refresh)`
- **优先级**：中（演示现场"显得真"的关键）

## 5. P3 — 一致性 / 复用（影响长期可维护）

### Issue-11：`被测系统` 下拉 computed 在 7 个页面拷贝 7 次
- **位置**：Connection / TestData / Task / Execution / Rule / Exception / Statistics 7 个 view
- **现状**：
  ```js
  const ALL_SYSTEM_VALUE = '__all__'
  const systemOptions = computed(() => systemStore.options.map((item) => ({
    ...item, selectValue: item.value == null ? ALL_SYSTEM_VALUE : item.value
  })))
  const systemSelectValue = computed({
    get: () => systemStore.currentId ?? ALL_SYSTEM_VALUE,
    set: (value) => systemStore.setCurrent(value === ALL_SYSTEM_VALUE ? null : value),
  })
  ```
- **建议改法**：抽 `src/composables/useSystemFilter.js`
  ```js
  export function useSystemFilter() {
    const systemStore = useSystemStore()
    const ALL = '__all__'
    const options = computed(() => systemStore.options.map((o) => ({
      ...o, selectValue: o.value == null ? ALL : o.value
    })))
    const value = computed({
      get: () => systemStore.currentId ?? ALL,
      set: (v) => systemStore.setCurrent(v === ALL ? null : v),
    })
    return { options, value }
  }
  ```
  各页 `const { options, value } = useSystemFilter()`
- **优先级**：中（修一次省 7 次维护）

## 6. P4 — 用户没明确要求但应该想到的视角

### 视角 1：可访问性（A11y）

- 大量按钮用 `el-button link` 没有文字只有图标（异常台账、规则表的操作列）
  - 修复：加 `aria-label`，或者用 `title` 属性
- 颜色是唯一的状态传达（在线=绿、离线=灰、告警=红）
  - 色盲用户无法分辨
  - 修复：加图标双通道（如 ⚠️ 红色、✓ 绿色）
- 表单错误信息只在 input 下方显示，screen reader 不一定能感知
  - 修复：加 `aria-describedby` 关联错误信息

### 视角 2：键盘 / 快捷键

联试现场操作密集（点 50+ 次才能完成一次任务），建议加：
- `Ctrl+1..9` 跳到对应模块
- `Ctrl+K` 全局命令面板（跳页 / 搜系统 / 搜任务 / 搜接口）
- `↑↓ ←→` 左右树导航
- `Enter` 打开选中节点
- `Delete` 删除（带确认）
- `Esc` 关闭弹窗
- `Ctrl+S` 保存当前编辑

### 视角 3：性能 / 体积

- `src/main.js:14-17` 把整个 `@element-plus/icons-vue` 全部 `app.component(key, component)`
  - 实际用到 ≤ 40 个
  - 全量注册让首屏多 ~300KB
  - 修复：按需引入
    ```js
    import { Plus, Search, Delete /* ... */ } from '@element-plus/icons-vue'
    app.component('IconPlus', Plus)
    // ...
    ```
- `src/mock/seed-data.js` 145KB — 已经是 mock，但如果接后端要注意
- `node_modules\.vite\deps\chunk-J75C6SMA.js` 等预构建包偏大，确认是不是按需

### 视角 4：演示场景的"流畅度"

- 执行监控是项目最吸睛的画面（实时滚动 HEX 报文 + 进度条 + 数字翻动）
- 完整跑 4 步大概 15-20 秒
- **建议**：加 0.5x / 1x / 2x 速度切换按钮，方便录屏和讲解
- 同时执行完成后没有 toast 通知 Dashboard —— 演示者要刷新页面才看到结果

### 视角 5：管理员 vs 测试员 角色差异

- admin：完整左栏 11 个图标 + 设置菜单里的"用户管理/权限管理"
- tester：只看到被授权的系统
- 这个权限过滤已实现（`useSystemStore.visibleSystems` getter）
- **但**：当前演示只有 admin 视角。**建议加角色切换演示脚本**：演示时切换"测试员 A" → "测试员 B"，让观众看到"同一份数据，不同用户看到不同"

### 视角 6：离线 / 单机版

- `vite-plugin-singlefile` 已在 `package.json` —— 准备打单文件
- 当前数据都在前端 store，刷新后大部分会丢
- **建议**：关键 store（系统/模块/协议/任务/规则）持久化到 localStorage
  ```js
  // stores/system.js
  actions: {
    init() {
      const saved = localStorage.getItem('jt_systems')
      if (saved) this.systems = JSON.parse(saved)
    },
    save() {
      localStorage.setItem('jt_systems', JSON.stringify(this.systems))
    }
  }
  // 调用所有 mutation 后 this.save()
  ```

### 视角 7：多人协同

- 8 个 store 没有**统一的变更通知**（mutation 时间戳、lastModified、updatedBy）
- 多人协同编辑同一份协议时会丢冲突
- 后续要接后端时，这一层要先补（CRDT 或 OT 算法）

### 视角 8：错误边界与降级

- 任何 store 抛错都会让白屏
- **建议**：在 main.js 加全局 errorHandler，关键页面加 ErrorBoundary
  ```js
  app.config.errorHandler = (err, instance, info) => {
    ElNotification.error({ title: '出错了', message: err.message })
    // 上报到 sentry / 自有日志
  }
  ```

### 视角 9：可演进性

- 7 个页面都"手画图表"（SVG / Canvas）
- 优点：体积小
- 缺点：图表种类受限（没有热力图、桑基图、漏斗图）
- 后期如果要做"接口响应热力图"或"模块依赖桑基图"，建议引入轻量 ECharts (gzipped ~200KB)
- 触发条件：演示后客户问"能再加 X 图吗"时再决定

### 视角 10：i18n / 主题

- 全部中文硬编码
- 演示对国内客户 OK，但项目规则写"国际"机构的话
- 短期：把 11 个页面的文案抽到 `src/locales/zh-CN.js`
- 长期：用 vue-i18n
